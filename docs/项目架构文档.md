# TradingAgents 项目架构详解

> 生成日期：2026-05-24 | 项目：LangGraph 多智能体交易分析系统（A 股适配版）

---

## 一、项目本质

一个 **LangGraph 多智能体交易分析系统**——模拟投资研究团队的完整工作流：从数据采集到最终买卖决策，中间经历多空辩论、风控审核。

## 二、入口（两个）

| 入口 | 用途 | 命令 |
|------|------|------|
| `main.py` | 快速测试脚本，接受命令行参数 | `.venv/bin/python main.py 600519` |
| `cli/main.py` | 完整交互式 CLI（typer + questionary + rich），8 步交互 | `.venv/bin/python -m cli.main analyze` |

**CLI 8 步流程**：选股票 → 选日期 → 选语言 → 选分析师组合 → 选辩论深度 → 选 LLM 供应商 → 选思考模型 → 选思考模式

---

## 三、5 阶段流水线

### Phase 1 — 分析师团队（4 人并行取数）

| 分析师 | 工具/方法 | 输出 |
|--------|-----------|------|
| 市场分析师 | `get_stock_data` + `get_indicators`（SMA/EMA/MACD/RSI/BOLL/ATR/VWMA） | `market_report` |
| 情绪分析师 | 预抓新闻 + StockTwits + Reddit（不走工具调用，直接注入 prompt） | `sentiment_report` |
| 新闻分析师 | `get_news` + `get_global_news` + `get_insider_transactions` | `news_report` |
| 基本面分析师 | **L1 财报引擎**（6 维 64 指标） | `fundamentals_report` |

**L1 引擎细节**：资本结构 / 资产质量 / 盈利能力 / 成长性 / 现金流 / 分红 → A/B/C/D 评级

### Phase 2 — 研究团队（多空辩论）

- **看多研究员** ← 4 份分析师报告 + 对方历史论点
- **看空研究员** ← 反驳看多论点
- 来回辩论 `max_debate_rounds` 轮（默认 1）
- **研究经理** 总结为结构化 `ResearchPlan`

### Phase 3 — 交易员

- ResearchPlan → `TraderProposal`
- 内容：Buy/Hold/Sell + 入场价 / 止损位 / 仓位大小

### Phase 4 — 风控团队（三方辩论）

- **激进辩手**：强调高风险高回报
- **保守辩手**：强调风险控制
- **中立辩手**：平衡分析
- 辩论 `max_risk_discuss_rounds` 轮（默认 1）

### Phase 5 — 组合经理（最终拍板）

- 综合所有信息 + 历史反思记忆 → `PortfolioDecision`
- 输出：5 级评级（Buy / Overweight / Hold / Underweight / Sell）+ 执行摘要 + 目标价

### 后处理

- `SignalProcessor.process_signal()` → 提取 5 级评级字符串
- `TradingMemoryLog.store_decision()` → 追加待定条目到 Markdown 日志
- 下次运行时：fetch 实际收益 → `Reflector.reflect_on_final_decision()` → 反思写入日志 → 注入下次 agent prompt

## 四、底层基础设施模块

| 模块 | 职责 | 关键文件 |
|------|------|----------|
| **dataflows/** | 数据获取层，路由器分发请求到不同数据源，支持自动降级 | `interface.py`, `akshare_data.py`, `y_finance.py`, `alpha_vantage*.py` |
| **llm_clients/** | LLM 客户端工厂，支持 12 个供应商 | `factory.py`, `model_catalog.py`, `openai_client.py`, `anthropic_client.py`, `google_client.py` |
| **graph/** | LangGraph 图编排：StateGraph 构建、条件边、断点续跑（SQLite）、决策反思 | `trading_graph.py`, `setup.py`, `conditional_logic.py`, `checkpointer.py`, `reflection.py` |
| **l1/** | A 股专用财报分析引擎 | `analyzer_l1_enhanced_complete.py`, `l1_akshare_fetcher.py`, `l1_config.py` |

---

## 五、核心设计模式

1. **供应商路由策略模式**：`interface.py` 的 `route_to_vendor()` 按优先级分发，自动降级（如 AV 限流 → fallback 到 yfinance）
2. **A 股自动检测**：`^\d{6}$` 正则匹配 → 自动切 akshare 数据源 + 上证指数基准
3. **结构化输出降级**：先尝试 Pydantic `with_structured_output()`，失败回退自由文本
4. **baostock 全局锁**：`_bs_lock` 串行化 TCP 会话，防止 8 并发互相干扰
5. **延迟导入**：LLM 客户端和 L1 引擎均采用 lazy import，避免启动时加载重型依赖

```
  │   │                                                   └── alpha_vantage*.py
  │   ├── researchers/*.py (多空辩论)
  │   ├── managers/*.py (研究经理+组合经理) → agents/schemas.py

  │   └── risk_mgmt/*.py (风控三方辩论)

  └── l1/l1_akshare_fetcher.py → L1FinancialAnalyzerEnhanced (6维64指标)
```

## 七、配置中心 (`default_config.py`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `llm_provider` | openai | LLM 供应商 |
| `deep_think_llm` | gpt-5.4 | 深度思考模型 |
| `quick_think_llm` | gpt-5.4-mini | 快速模型 |
| `max_debate_rounds` | 1 | 多空辩论轮数 |
| `max_risk_discuss_rounds` | 1 | 风控辩论轮数 |
| `data_vendors` | akshare | 数据供应商 |
| `benchmark_map` | 按交易所后缀 | 自动选基准指数（A股→上证指数） |

支持环境变量覆盖：`TRADINGAGENTS_*` 前缀的环境变量自动覆盖对应配置项。
